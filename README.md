Basic DNS Management with Route 53 Simple Routing

Project Overview

This project demonstrates how to configure a Simple Routing Policy in AWS Route 53. This policy is used to map a domain name 
(like example.com) to a single resource, such as an IP address or an AWS resource (ELB, S3 bucket). It is the foundational building block for managing DNS records in the cloud.

Prerequisites

An active AWS Account.

A domain name to use for the Hosted Zone.

The IP address or DNS name of the destination resource (e.g., an EC2 instance).
